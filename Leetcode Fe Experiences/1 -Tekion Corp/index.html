Reflow v/s Repaint in Rendering

https://medium.com/swlh/what-the-heck-is-repaint-and-reflow-in-the-browser-b2d0fb980c08

Reflow

When it happens: Reflow is triggered by changes that affect the layout of the document. For example:

  • Adding or removing elements from the DOM
  • Changing the size (width, height) or position of elements.
  • Modifying properties like padding, border, margin, or display.
  • Chaning the font sizes
  • left or top

Repaint

When it happens: Repaint is triggered by changes to visual styles that do not require a recalculation of the layout, such as:

  • - >>Changing color, background-color, visibility, or outline.
  • - >>Modifying box-shadow, or border-color.

    • Optimizations:
    Minimize layout changes (reflow) by batching DOM changes or using CSS properties that only trigger repaints. Avoid frequent or unnecessary changes that trigger both reflow and repaint in critical sections of the page (e.g., during scroll or animations). Use strategies like requestAnimationFrame for smoother animations.
